BufferAttribute

Данный класс сохраняет данные атрибута (такие как положения вершин, индексы граней, нормали, цвета, текстурные координаты (UV) и все атрибуты, установленные пользователем), связанного с BufferGeometry, который позволяет более эффективное прохождение всех этих данных к GPU. Посмотрите эту страницу для дополнительных сведений и примеров использования.

Данные хранятся как векторы произвольной длины (определяемые itemSize), и вообще в методах, описанных ниже, если передаются с индексом, то он автоматически умножается на длину вектора.

Конструктор

BufferAttribute( array, itemSize, normalized )
array - должен быть TypedArray (типизированным массивом). Используется для создания экземпляра буфера.
В этом массиве должно быть (itemSize * numVertices) элементов, где numVertices - это число вершин в соответствующей BufferGeometry.
itemSize - число значений массива, связанного с конкретной вершиной. Например, если данный атрибут хранит 3-компонентный вектор (такой как position - положение, normal - нормаль, или color - цвет), тогда itemSize должен быть равен 3.
normalized - дополнительный, необязательный атрибут, применяется только для целочисленных данных и показывает как базовые данные в буфере сопоставлены со значениями в коде GLSL. Например, если массив (аргумент array) является экземпляром UInt16Array, а аргумент normalized установлен как true, то значения от 0 до +65535 в данных массива будут отображены в атрибуте GLSL как значения от 0.0f до +1.0f. Массив Int16Array (со знаком) от -32767 до +32767 будет сопоставлен как -1.0f - +1.0f. Если normalized установлен как false значения будут конвертированы в числа с плавающей запятой, которые содержат точное значение, т.е. 32767 станет 32767.0f.

Свойства


.array
Массив, в котором содержаться данные, хранящиеся в буфере.

.count
Хранит длину массива деленную на itemSize.
Если буфер хранит 3-компонентный вектор (как например положение, нормали или цвет), то в этом случае будет подсчитано число таких хранящихся векторов.

.isBufferAttribute
Используется для проверки является ли данный класс или производные от него BufferAttribute. Значение по умолчанию равно true.
Не изменяйте это свойство, так как оно используется для оптимизации внутри three.js.

.itemSize
Длина векторов, хранящихся в array.

.name
Дополнительное имя для экземпляра данного атрибута. По умолчанию это пустая строка.

.needsUpdate
Флаг, показывающий что данный атрибут был изменен и должен быть повторно отправлен в графический процессор (GPU). Установите его как true при изменении значения массива.
Установка этого значения как true также увеличивает version.

.normalized
Указывает каким образом базовые данные в буфере соответствуют значениям в коде GLSL. Подробности смотрите выше, в описании конструктора.

.onUploadCallback
Функция обратного вызова, выполняемая после того, как визуализатор (renderer) передал данные массива атрибутов в GPU.

.updateRange
Объект содержит:
offset: позиция, с которой начинается обновление, значение по умолчанию равно 0.
count: значение по умолчанию равно -1, что означает не использовать диапазон обновления.
Данное свойство может быть использовано для отдельного обновления некоторых компонентов хранящихся векторов (например, только для компонента относящегося к цвету).

.usage
Определяет предполагаемый шаблон использования хранилища данных в целях оптимизации. Соответствует параметру usage в WebGLRenderingContext.bufferData(). Значением по умолчанию является THREE.StaticDrawUsage.

usage
Значение GLenum указывающее предполагаемую схему использования хранилища данных в целях оптимизации. Возможные значения:
gl.STATIC_DRAW: содержимое предназначено для того, чтобы устанавливаться приложением только один раз и многократно использоваться как источник команд для WebGL рисования и описания изображения.
gl.DYNAMIC_DRAW: содержимое предназначено для того, чтобы повторно переопределяться приложением и многократно использоваться как источник команд для WebGL рисования и описания изображения.
gl.STREAM_DRAW: содержимое предназначено для того, чтобы устанавливаться приложением только один раз и использоваться не более чем несколько раз в качестве источника команд для WebGL рисования и описания изображения.
При использовании среды WebGL 2 дополнительно доступны следующие значения:
gl.STATIC_READ: предполагается что содержимое будет установлено только однажды, считыванием данных из WebGL, и многократно запрашиваться приложением.
gl.DYNAMIC_READ: содержимое предназначено для того, чтобы повторно переопределяться, считыванием данных из WebGL, и многократно запрашиваться приложением.
gl.STREAM_READ: содержимое предназначено для того, чтобы однократно определиться, считыванием данных из WebGL, и запрашиваться приложением не более чем несколько раз.
gl.STATIC_COPY: содержимое предназначено для того, чтобы однократно определиться, считыванием данных из WebGL, и многократно использоваться в качестве источника команд для WebGL рисования и описания изображения.
gl.DYNAMIC_COPY: содержимое предназначено для того, чтобы повторно переопределяться, считыванием данных из WebGL, и многократно использоваться в качестве источника команд для WebGL рисования и описания изображения.
gl.STREAM_COPY: содержимое предназначено для того, чтобы однократно определиться, считыванием данных из WebGL, и использоваться не более чем несколько раз в качестве источника команд для WebGL рисования и описания изображения.

.version
Номер версии, увеличивающийся каждый раз, когда свойство needsUpdate устанавливается как true.

Методы


.applyMatrix3 ( m : Matrix3 ) : this
Applies matrix m to every Vector3 element of this BufferAttribute.

.applyMatrix4 ( m : Matrix4 ) : this
Applies matrix m to every Vector3 element of this BufferAttribute.

.applyNormalMatrix ( m : Matrix3 ) : this
Applies normal matrix m to every Vector3 element of this BufferAttribute.

.transformDirection ( m : Matrix4 ) : this
Applies matrix m to every Vector3 element of this BufferAttribute, interpreting the elements as a direction vectors.

.clone( )
Возвращает копию данного bufferAttribute.

.copy( bufferAttribute )
Копирует в данный BufferAttribute другой (указанный в аргументе) BufferAttribute.

.copyArray( array )
Копирует, заданный здесь в качестве аргумента, массив (который может быть обычным массивом или типизированным - TypedArray) в массив array. Если копируется типизированный массив (TypedArray), ознакомьтесь на TypedArray.set с примечаниями по требованиям. Вот TypedArray.set на русском языке.

.copyAt( index1, bufferAttribute, index2 )
Копирует вектор из bufferAttribute[index2] в array[index1].

.copyColorsArray( colors )
Копирует массив, представленный значениями цвета в формате RGB, в array.

.copyVector2sArray( vectors )
Копирует массив, представленный векторами Vector2, в array.

.copyVector3sArray( vectors )
Копирует массив, представленный векторами Vector3, в array.

.copyVector4sArray( vectors )
Копирует массив, представленный векторами Vector4, в array.

.getX( index )
Возвращает компоненту x вектора по заданному индексу.

.getY( index )
Возвращает компоненту y вектора по заданному индексу.

.getZ( index )
Возвращает компоненту z вектора по заданному индексу.

.getW( index )
Возвращает компоненту w вектора по заданному индексу.

.onUpload ( callback )
Устанавливает значение свойства onUploadCallback.
В примере WebGL / Buffergeometry этот метод используется для высвобождения памяти после того, как буфер был передан GPU.

.set( value, offset )
value - массив или типизированный массив, из которого копируются значения.
offset - индекс массива array, с которого начинается копирование, (дополнительный, необязательный аргумент).
Метод вызывает метод TypedArray.set( value, offset ) на данный array.
В частности, посмотрите эту страничку насчет требований к value в TypedArray. Вот здесь она на русском языке.

.setUsage( value )
Метод устанавливает значение, переданное в аргументе value, как значение свойства usage.

.setX( index, x )
Устанавливает компонент x вектора по заданному индексу.

.setY( index, y )
Устанавливает компонент y вектора по заданному индексу.

.setZ( index, z )
Устанавливает компонент z вектора по заданному индексу.

.setW( index, w )
Устанавливает компонент w вектора по заданному индексу.

.setXY( index, x, y )
Устанавливает компоненты вектора x, и y по заданному индексу.

.setXYZ( index, x, y, z )
Устанавливает компоненты вектора x, y, и z по заданному индексу.

.setXYZW( index, x, y, z, w )
Устанавливает компоненты вектора x, y, z и w по заданному индексу.

Типы BufferAttribute

В three.js имеются девять типов BufferAttribute. Они соответствуют типизированным массивам в JavaScript.

Конструктор

Все вышеперечисленное вызывается одним и тем же способом. 
TypedBufferAttribute( array, itemSize, normalized )
array - это может быть типизированный или нетипизированный (обычный) массив или целое число в качестве длины массива. Значение массива будет преобразовано в массив указанного типа. Если указана длина массива, будет создан новый типизированный массив (TypedArray), который инициализируется со всеми элементами, установленными как "ноль".
itemSize - число значений массива, которые должны быть увязаны с конкретной вершиной.
normalized - дополнительный, необязательный атрибут, который показывает как базовые данные в буфере сопоставлены со значениями в коде GLSL.
