OBJLoader2

Загрузчик для загрузки ресурсов в формате obj.
Формат файла OBJ - это простой формат данных, представляющий трехмерную геометрию в удобочитаемой для человека форме, в виде положения каждой вершины, UV-координат позиции каждой текстуры, связанных с вершиной, нормалей вершин, граней, образуемых каждым многоугольником, определяемым списком вершин, и вершин текстур.
Статья Википедии о формате obj на русском языке.

Пример

// instantiate the loader (создание экземпляра загрузчика)
let loader = new THREE.OBJLoader2();

// function called on successful load (вызывается по успешному завершению загрузки)
function callbackOnLoad ( object3d ) {
  scene.add( object3d );
};

// load a resource from provided URL synchronously
// синхронная загрузка ресурса из предоставленного URL-адреса
loader.load( 'obj/female02/female02.obj', callbackOnLoad, null, null, null );
Другие примеры:
webgl/loader/obj2 - простой пример
webgl/loader/obj2/options - пример нескольких вариантов использования (разбор, загрузка
                            и запуск с инструкциями (синхронно или параллельно основному
                            - смотрите описание OBJLoader2Parallel))

Конструктор

OBJLoader2( manager )
manager — менеджер загрузки, используемый для загрузки. Значением по умолчанию является THREE.DefaultLoadingManager.
Создает новый OBJLoader2. Используйте OBJLoader2 для загрузки данных в формате OBJ из файлов или для синтаксического анализа OBJ-данных из текста или буферного массива.

Методы

Общие методы смотрите в описании базового класса Loader. 

.parse( content )
content - данные в формате OBJ или в виде массива из 8-разрядных целых чисел без знака (Uint8Array), или в виде строкового значения (string).
Метод проводит синхронный синтаксический анализ данных в формате OBJ из буферного массива или строкового значения и возвращает базовый трехмерный объект (baseObject3d).
Примечание переводчика: Базовый трехмерный объект - это объект, уже прикрепленный к графу сцены, к которому будут присоединены новые сетки (mesh), смотрите метод setBaseObject3d.

.load( url, onLoad, onProgress, onError, onMeshAlter )
url — строка, содержащая путь/URL-адрес загружаемого файла.
onLoad — функция, которая будет вызвана после успешного завершения загрузки. В качестве аргумента, эта функция получает загруженный Object3D.
onProgress - функция, которая будет вызываться пока идет процесс загрузки (дополнительный, необязательный параметр). Аргументом будет экземпляр XMLHttpRequest, что содержит сведения о загрузке total и loaded в байтах.
onError — функция, которая будет вызвана в случае ошибки при загрузке (дополнительный, необязательный параметр). Функция принимает ошибку в качестве аргумента.
onMeshAlter - функция, которая будет вызвана после того, как новые исходные данные сетки станут доступными для изменения. Дополнительный, необязательный параметр.
Используйте этот удобный метод для загрузки файла по указанному URL-адресу. По умолчанию загрузчик файлов использует буферный массив.

.setLogging( enabled, debug )
enabled - логическое значение (true или false).
debug - логическое значение (true или false).
Метод включает или отключает логирование (ведение журнала) в целом (за исключением предупреждений и ошибок), а также включает или отключает ведение журнала отладки.

.addMaterialPerSmoothingGroup( materialPerSmoothingGroup )
materialPerSmoothingGroup - логическое значение (true или false).
Метод указывает, должен ли материал создаваться для группы сглаживания.

.setUseOAsMesh( useOAsMesh )
useOAsMesh - логическое значение (true или false).
Обычно "o" является мета-информацией и не приводит к созданию новых сеток, но возможно принудительное создание сетки (mesh) при появлении "o".
Примечание переводчика: Здесь "o" - это именованный объект смотрите раздел "Справочные материалы" статьи Wavefront .obj file в англ. Википедии.

.setUseIndices( useIndices )
useIndices - логическое значение.
Метод поручает загрузчикам создавать индексированную BufferGeometry.

.setDisregardNormals( disregardNormals )
disregardNormals - логическое значение.
Метод указывает, следует ли полностью проигнорировать и восстановить нормали.

.setModelName( modelName )
modelName - название модели.
Метод устанавливает название (имя) модели.

.setBaseObject3d( baseObject3d )
baseObject3d - объект, уже прикрепленный к графу сцены, к которому будут присоединены новые сетки (mesh).
Метод устанавливает узел, к которому будут непосредственно присоединены загруженные объекты.

.setMaterials( materials )
materials - массив материалов.
Добавляет материалы как ассоциированный (связанный) массив.

.setCallbackOnLoad( onLoad )
onLoad - функция, вызываемая при загрузке
Метод регистрирует функцию, которая вызывается по завершении синтаксического разбора.

.setCallbackOnAssetAvailable( onAssetAvailable )
onAssetAvailable - функция, вызываемая когда ресурс становится доступным.
Метод регистрирует функцию, которая вызывается как только ресурс (сетка или материал) становится доступным.

.setCallbackOnProgress( onProgress )
onProgress - функция, вызываемая во время процесса.
Метод регистрирует функцию, которая используется для сообщений об общем ходе обработки.

.setCallbackOnError( onError )
onError - функция, вызываемая в случае ошибки процесса обработки.
Метод регистрирует функцию обработчика ошибок, которая вызывается в случае ошибки процесса обработки. Она может решить или вести журнал, или выдать исключение.

.setCallbackOnMeshAlter( onMeshAlter )
onMeshAlter - функция, вызываемая как только сетка становится доступной.
Метод регистрирует функцию, которая вызывается как только сетка становится доступной и она может быть изменена предоставленной функцией.
