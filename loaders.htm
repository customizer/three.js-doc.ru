Загрузчики

В этом разделе рассматриваются различные загрузчики, используемые для загрузки изображений, материалов, текстур, шрифта и т.д.
AnimationLoader - класс для загрузки клипов анимации.
BufferGeometryLoader - загрузчик для загрузки BufferGeometry.
CompressedTextureLoader - базовый класс для загрузчика блочных текстур.
DataTextureLoader - базовый класс для загрузки типовых двоичных форматов текстур.
FontLoader - класс для загрузки шрифта в JSON-формате
ImageLoader - класс для загрузки изображений.
Loader - базовый класс для реализации загрузчиков.
MaterialLoader - класс для загрузки материалов в формате JSON.
TextureLoader - класс для загрузки текстуры.
AudioLoader - класс для загрузки AudioBuffer.
Cache - система кэширования, используемая внутри FileLoader.
CubeTextureLoader - класс для загрузки кубической текстуры.
FileLoader - низкоуровневый класс для загрузки ресурсов.
ImageBitmapLoader - загрузчик для изображений в виде ImageBitmap.
LoaderUtils - некоторые вспомогательные загрузочные функции.
ObjectLoader - загрузка ресурсов в формате JSON.

Менеджеры загрузки
      DefaultLoadingManager - менеджер загрузки, применяемый по умолчанию.
      LoadingManager - обрабатывает и отслеживает загружаемые и ожидающие загрузки данные.

AnimationLoader

Класс для загрузки клипов анимации в формате JSON. Для загрузки файлов, внутри класса, используется FileLoader.

Пример

// instantiate a loader (создаем экземпляр загрузчика)
var loader = new THREE.AnimationLoader();

// load a resource (загрузка ресурса)
loader.load(
  // resource URL(URL-адрес ресурса)
  'animations/animation.js',

  // onLoad callback (функция обратного вызова, вызываемая после загрузки)
  function ( animations ) {
    // animations is an array of AnimationClips
    // анимация - это массив AnimationClips
  },

  // onProgress callback (функция обратного вызова, вызываемая в процессе загрузки)
  function ( xhr ) {
    console.log( (xhr.loaded / xhr.total * 100) + '% загружено' );
  },

  // onError callback (функция обратного вызова, вызываемая при ошибке)
  function ( err ) {
    console.log( 'Произошла ошибка' );
  }
);

Конструктор

AnimationLoader( manager )
manager — менеджер загрузки, используемый загрузчиком. Значением по умолчанию является THREE.DefaultLoadingManager.
Создает новый AnimationLoader.

Свойства


.manager
Менеджер загрузки, используемый загрузчиком. Значением по умолчанию является THREE.DefaultLoadingManager.

Методы


.load( url, onLoad, onProgress, onError )
url — путь или URL к файлу. Также это может быть Data URI.
onLoad — функция, которая будет вызвана по завершении загрузки. Аргументом будет загруженная анимация.
onProgress — функция, которая будет вызываться пока идет процесс загрузки. Аргументом будет экземпляр XMLHttpRequest, что содержит сведения о загрузке total и loaded в байтах.
onError — функция, которая будет вызвана в случае ошибки при загрузке.
Начинает загрузку с URL-адреса и передает загруженную анимацию функции, указанной в onLoad.

.parse( json, onLoad )
json — обязателен
onLoad — функция, вызываемая по завершении разбора (парсинга) JSON объекта.
Анализирует (разбирает) объект JSON и передает результаты функции, приведенной в аргументе onLoad. Отдельные клипы объекта будут обработаны с помощью метода AnimationClip.parse.

AudioLoader

Класс для загрузки AudioBuffer. Для загрузки файлов, внутри класса, используется FileLoader.

Пример

// instantiate a listener (создаем экземпляр слушателя)
var audioListener = new THREE.AudioListener();

// add the listener to the camera (добавляем слушателя к камере)
camera.add( audioListener );

// instantiate audio object (создаем экземпляр аудиообъекта)
var oceanAmbientSound = new THREE.Audio( audioListener );

// add the audio object to the scene (добавляем к сцене аудиообъект)
scene.add( oceanAmbientSound );

// instantiate a loader (создаем экземпляр загрузчика)
var loader = new THREE.AudioLoader();

// load a resource (загружаем ресурс)
loader.load(
  // resource URL (адрес ресурса)
  'audio/ambient_ocean.ogg',
  // onLoad callback
  // функция обратного вызова, вызываемая после загрузки ресурса
  function ( audioBuffer ) {
    // set the audio object buffer to the loaded object
    // устанавливаем буфер аудиообъекта для загруженного объекта
    oceanAmbientSound.setBuffer( audioBuffer );

    // play the audio - воспроизводим аудио
    oceanAmbientSound.play();
  },
  // onProgress callback
  // функция обратного вызова, вызываемая во время загрузки
  function ( xhr ) {
    console.log( (xhr.loaded / xhr.total * 100) + '% загружено' );
  },
  // onError callback
  // функция обратного вызова, вызываемая при ошибках загрузки
  function ( xhr ) {
    console.log( 'Произошла ошибка' );
  }
);

BufferGeometryLoader

Загрузчик для загрузки BufferGeometry. Для загрузки файлов, внутри класса, используется FileLoader.

Пример

// instantiate a loader (создаем экземпляр загрузчика)
var loader = new THREE.BufferGeometryLoader();

// load a resource (загрузка ресурса)
loader.load(
	// resource URL (URL-адрес ресурса)
	'models/json/pressure.json',
	// Function when resource is loaded
    // функция, вызываемая при загрузке ресурса
	function ( geometry ) {
		var material = new THREE.MeshLambertMaterial( { color: 0xF5F5F5 } );
		var object = new THREE.Mesh( geometry, material );
		scene.add( object );
	},
	// Function called when download progresses
    // функция, вызываемая при процессе загрузки
	function ( xhr ) {
		console.log( (xhr.loaded / xhr.total * 100) + '% загружено' );
	},
	// Function called when download errors
    // функция, вызываемая при ошибке загрузки
	function ( xhr ) {
		console.log( 'An error happened' );
	}
);

Cache

Простая система кэширования, используемая внутри FileLoader.

Примеры

WebGL / geometry / text
WebGL / interactive / instances / gpu
WebGL / loader / ttf

Применение

Чтобы включить кэширование во всех загрузчиках, использующих FileLoader, установите 
THREE.Cache.enabled = true

Свойства


.enabled
Свойство определяет, будет ли включено кэширование. Значением по умолчанию является false.

.files
Объект, удерживающий кэшированные файлы.

Методы


.add( key, file )
key — строковый ключ, который будет связан с кэшируемым файлом.
file — файл, который нужно кэшировать.
Метод добавляет запись в кэш с ключом, ссылающимся на файл. Если по этому ключу уже содержится файл, то он перезаписывается.

.get( key )
key — строковый ключ.
Метод получает значение key. Если ключа не существует, возвращается значение undefined, т.е. неопределено.

.remove( key )
key — строковый ключ, связанный с кэшированным файлом.
Удаляет кэшированный файл, связанный с данным ключом.

.clear( )
Удаляет все значения из кэша.

FileLoader

Низкоуровневый класс для загрузки ресурсов с помощью XmlHttpRequest, используемый большинством загрузчиков.

Пример

var loader = new THREE.FileLoader();

// load a text file and output the result to the console
// загружаем текстовый файл и выводим результат в консоли
loader.load(
  // resource URL (URL-адрес ресурса)
  'example.txt',

  // onLoad callback (функция обратного вызова, вызываемая по окончании загрузки)
  function ( data ) {
    // output the text to the console (вывод результата в консоль)
    console.log( data )
  },

  // onProgress callback (функция обратного вызова, вызываемая в процессе загрузки)
  function ( xhr ) {
    console.log( (xhr.loaded / xhr.total * 100) + '% загружено' );
  },

  // onError callback (функция обратного вызова, вызываемая при ошибке)
  function ( err ) {
    console.error( 'Произошла ошибка' );
  }
);
Примечание: кэш должен быть включен с помощью 
THREE.Cache.enabled = true;
Это глобальное свойство и задается только один раз для использования всеми загрузчиками, которые внутри себя применяют FileLoader. Кэш это кэширующий модуль, который удерживает ответ от каждого запроса, сделанного через данный загрузчик, так что каждый файл запрашивается только однажды.

TextureLoader

Класс для загрузки текстуры. Он внутренне использует для загрузки файлов ImageLoader.

Пример

var texture = new THREE.TextureLoader().load( 'textures/land_ocean_ice_cloud_2048.jpg' );

// immediately use the texture for material creation
// сразу же используем текстуру для создания материала
var material = new THREE.MeshBasicMaterial( { map: texture } );

Пример с функцией обратного вызова

// instantiate a loader (создаем экземпляр загрузчика)
var loader = new THREE.TextureLoader();

// load a resource (загружаем ресурс)
loader.load(
  // resource URL (URL-адрес ресурса
  'textures/land_ocean_ice_cloud_2048.jpg',

  // onLoad callback (функция обратного вызова на окончание загрузки)
  function ( texture ) {
    // in this example we create the material when the texture is loaded
    // в этом примере материал создается по окончании загрузки текстуры
    var material = new THREE.MeshBasicMaterial( {
      map: texture
    } );
  },

  // onProgress callback currently not supported
  // функция обратного вызова во время процесса в настоящее время не поддерживается)
  undefined,

  // onError callback (функция обратного вызова при ошибке)
  function ( err ) {
    console.error( 'Произошла ошибка.' );
  }
);
Обратите внимание, что версия three.js r84 для ImageLoader прекратила поддержку событий на ход процесса (on progress). Для TextureLoader с поддержкой событий ход процесса, смотрите эту тему.

Свойства


.crossOrigin
Если значение свойства установлено, то перед началом загрузки атрибуту изображения crossOrigin присваивается значение данного свойства crossOrigin. Значением по умолчанию является undefined, т.е. значение не определено.

.manager
Менеджер загрузки, используемый загрузчиком. Значением по умолчанию является DefaultLoadingManager.

.path
Базовый путь, из которого будут загружаться текстуры. Смотрите описание метода setPath. Значением по умолчанию является undefined, т.е. значение не определено.

.withCredentials
Свойство определяет, использует ли XMLHttpRequest учетные данные. Смотрите описание метода setWithCredentials. Значением по умолчанию является undefined, т.е. значение не определено.

Методы


.load( url, onLoad, onProgress, onError )
url — путь или URL к файлу (обязательный аргумент). Также это может быть Data URI.
onLoad — функция, которая будет вызвана по завершении загрузки. Аргументом будет загруженная текстура.
onProgress — функция, которая будет вызываться пока идет процесс загрузки. Аргументом будет экземпляр XMLHttpRequest, что содержит сведения о загрузке total и loaded в байтах.
onError — функция, которая будет вызвана в случае ошибки при загрузке.
Метод начинает загрузку с указанного url-адреса (аргумент url) и передает загруженную текстуру функции, указанной в аргументе onLoad. Также метод возвращает новый объект текстуры, который может быть использован непосредственно для создания материала. Если это так и происходит, текстура может появиться на сцене после завершения соответствующего процесса загрузки.

.setCrossOrigin( value )
Устанавливает атрибут crossOrigin.

.setPath( path )
Метод устанавливает базовый путь или URL-адрес, с которого загружаются файлы. При загрузке множества моделей из одного и того же адреса, он может быть пригодиться.

.setWithCredentials( value )
Метод устанавливает, будет ли XMLHttpRequest использовать учетные данные, такие как куки, заголовки авторизации или клиентские сертификаты TLS. Смотрите описание XMLHttpRequest.withCredentials.
Обратите внимание, что метод не действует, если файлы загружаются локально или из одного и того же домена.

Менеджеры загрузки

DefaultLoadingManager

Общий экземпляр LoadingManager, используемый большинством загрузчиков, когда не указывается отдельный, специализированный менеджер загрузки.
В большинстве случаем его должно быть достаточно, однако иногда могут потребоваться отдельные менеджеры загрузки для текстур и моделей.

Пример

Для менеджера загрузки можно дополнительно устанавливать функции onStart, onLoad, onProgress, onError. Затем они будут применяться для любых загрузчиков, используемых DefaultLoadingManager.
Обратите внимание, что не следует путать эти функции с функциями с такими же названиями отдельных, специализированных загрузчиков, поскольку они предназначены для отображения информации об общем состоянии загрузки, а не для обработки загружаемых данных. 
THREE.DefaultLoadingManager.onStart = function ( url, itemsLoaded, itemsTotal ) {
  console.log( 'Начата загрузка файлов: '+url+'.\nЗагружено '+itemsLoaded+' из '+itemsTotal+' файлов.' );
};

THREE.DefaultLoadingManager.onLoad = function ( ) {
  console.log( 'Загрузка завершена!');
};

THREE.DefaultLoadingManager.onProgress = function ( url, itemsLoaded, itemsTotal ) {
  console.log( 'Загрузка файлов: '+url+'.\nЗагружено '+itemsLoaded+' из '+itemsTotal+' файлов.' );
};

THREE.DefaultLoadingManager.onError = function ( url ) {
  console.log( 'Произошла ошибка при загрузке '+url );
};

LoadingManager

Обрабатывает и отслеживает загружаемые и ожидающие загрузки данные. Если он не предоставлен вручную, то глобальный экземпляр данного класса создается и используется загрузчиками по умолчанию - смотрите DefaultLoadingManager.
В принципе его должно быть достаточно, однако бывают случаи, когда полезно иметь раздельные загрузчики,- например если требуется показать отдельные индикаторы загрузки для объектов и текстур.

Пример

Этот пример показывает способ использования LoadingManager'а для отслеживания процесса работы загрузчика OBJLoader. 
var manager = new THREE.LoadingManager();
manager.onStart = function ( url, itemsLoaded, itemsTotal ) {
  console.log( 'Начата загрузка файлов: '+url+'.\nЗагружено '+itemsLoaded+' из '+itemsTotal+' файлов.' );
};

manager.onLoad = function ( ) {
  console.log( 'Загрузка завершена!');
};

manager.onProgress = function ( url, itemsLoaded, itemsTotal ) {
  console.log( 'Загрузка файлов: '+url+'.\nЗагружено '+itemsLoaded+' из '+itemsTotal+' файлов.' );
};

manager.onError = function ( url ) {
  console.log( 'Произошла ошибка при загрузке '+url );
};

var loader = new THREE.OBJLoader( manager );
loader.load( 'file.obj', function ( object ) {
  //
} );
В дополнение к наблюдению за ходом процесса, LoadingManager можно использовать для переопределения во время загрузки URL-адресов ресурсов. Это может быть полезно для элементов, поступающих из событий перетаскивания, WebSockets, WebRTC или других API.
Вот пример, показывающий как загрузить модель в память используя URL-адрес объекта Blob. 
// Blob or File objects created when dragging files into the webpage.
// объекты Blob или File созданные при перетаскивании файлов на веб-страницу.
var blobs = {'fish.gltf': blob1, 'diffuse.png': blob2, 'normal.png': blob3};

var manager = new THREE.LoadingManager();

// Initialize loading manager with URL callback.
// инициализация менеджера загрузки с функцией обратного вызова для URL-адреса.
var objectURLs = [];
manager.setURLModifier( ( url ) => {
  url = URL.createObjectURL( blobs[ url ] );
  objectURLs.push( url );
  return url;
} );

// Load as usual, then revoke the blob URLs.
// загружаем как обычно, затем отменяем URL-адреса blob.
var loader = new THREE.GLTFLoader( manager );
loader.load( 'fish.gltf', (gltf) => {
  scene.add( gltf.scene );
  objectURLs.forEach( ( url ) => URL.revokeObjectURL( url ) );
});

Конструктор

LoadingManager( onLoad, onProgress, onError )
onLoad — эта функция будет вызываться по окончании работы всех загрузчиков. Дополнительный, необязательный аргумент.
onProgress — эта функция будет вызываться по завершении работы каждого отдельного элемента загрузки. Дополнительный, необязательный аргумент.
onError — эта функция будет вызываться при ошибках загрузчика. Дополнительный, необязательный аргумент.
Создает новый LoadingManager.

Свойства


.onStart
Данная функция будет вызвана при запуске загрузки.
Аргументами являются:
url — url-адрес загружаемого элемента.
itemsLoaded — число элементов, уже загруженных в настоящее время.
itemsTotal — полное число загружаемых элементов.
По умолчанию она не определена, т.е. undefined.

.onLoad
Данная функция будет вызвана по завершении всех загрузок. По умолчанию она не определена (т.е. undefined), если не передается в конструкторе.

.onProgress
Данная функция будет вызвана по завершении загрузки элемента.
Аргументами являются:
url — url-адрес загруженного элемента.
itemsLoaded — число элементов, уже загруженных в настоящее время.
itemsTotal — полное число загружаемых элементов.
По умолчанию она не определена (т.е. undefined), если не передается в конструкторе.

.onError
Данная функция будет вызвана при ошибках с любым элементом. Аргументом функции является:
url — url-адрес элемента с ошибкой.
По умолчанию она не определена (т.е. undefined), если не передается в конструкторе.

Методы


.setURLModifier( callback )
callback — функция обратного вызова модификатора url-адреса. Вызывается с url-адресом в качестве аргумента и должна возвращать урегулированный url-адрес.
Если аргумент предоставляется, то перед отправкой запроса функция обратного вызова будет переправлять каждый url-адрес ресурса. Функция обратного вызова может возврашать либо исходный, либо новый url-адрес для переопределения поведения загрузки. Такое поведение можно использовать для загрузки ресурсов из ZIP-файлов, при перетаскивании с применением API и Data URI.

.resolveURL( url )
url — url-адрес загрузки.
Метод задает url-адрес с использованием функции обратного вызова модификатора URL-адреса (если таковая имеется) и возвращает урегулированный url-адрес. Если модификатор url-адреса не установлен, возвращается исходный url-адрес.

Примечание: Следующие методы предназначены для внутреннего вызова загрузчиками, напрямую вызывать их нельзя.

.itemStart( url )
url — url-адрес загрузки.
Данный метод будет вызываться любым загрузчиком, которого использует менеджер, при старте загрузки с url-адреса.

.itemEnd( url )
url — url-адрес загрузки.
Данный метод будет вызываться любым загрузчиком, которого использует менеджер, по завершении загрузки с url-адреса.

.itemError( url )
url — url-адрес загрузки.
Данный метод будет вызываться любым загрузчиком, которого использует менеджер, когда происходит ошибка при загрузке с url-адреса.
