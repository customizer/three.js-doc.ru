CubeTextureLoader

Класс для загрузки кубической текстуры. Для загрузки файлов, внутри класса, используется ImageLoader.

Пример

var scene = new THREE.Scene();
scene.background = new THREE.CubeTextureLoader()
  .setPath( 'textures/cubeMaps/' )
  .load( [
    'px.png',
    'nx.png',
    'py.png',
    'ny.png',
    'pz.png',
    'nz.png'
  ] );

Конструктор

CubeTextureLoader( manager )
manager — менеджер загрузки, используемый загрузчиком. Значением по умолчанию является THREE.DefaultLoadingManager.
Создает новый CubeTextureLoader.

Методы

Смотрите общие методы в описании базового класса Loader. 

.load( urls, onLoad, onProgress, onError )
urls — массив из шести url-адресов к изображениям, по одному на каждую из сторон CubeTexture. URL-адреса должны указываться в следующем порядке: pos-x, neg-x, pos-y, neg-y, pos-z, neg-z. Также это может быть Data URI.
Обратите внимание что по соглашению, карты кубической текстуры задаются по системе координат, в которой положительное значение по оси x находится справа, если смотреть со стороны положительной оси z, - другими словами, используется левосторонняя система координат. Поскольку three.js использует правостороннюю систему координат, карты окружения, используемые в three.js, поменяют местами положительную и отрицательную оси координат x.
onLoad — функция, которая будет вызвана по завершении загрузки. Аргументом будет загруженная текстура.
onProgress — функция, которая будет вызываться пока идет процесс загрузки. Аргументом будет экземпляр XMLHttpRequest, что содержит сведения о загрузке total и loaded в байтах.
onError — функция, которая будет вызвана в случае ошибки при загрузке.
Метод начинает загрузку из источника (аргумент urls) и передает загруженную текстуру в функцию, определенную в аргументе onLoad.
